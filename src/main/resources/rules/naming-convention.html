Flow names must follow kebab-case naming convention

Flow names must use lowercase kebab-case format (e.g., 'get-customer-details', 'process-order'). This pattern matches lowercase letters, colons, curly braces, and hyphens. Consistent naming improves code readability and maintainability.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="getCustomerDetails"><logger/></flow>
  <flow name="ProcessOrder"><logger/></flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="get-customer-details"><logger/></flow>
  <flow name="process-order"><logger/></flow>
</mule>
