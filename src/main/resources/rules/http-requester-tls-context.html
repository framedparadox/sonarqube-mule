HTTP Request must configure TLS context for secure connections

HTTP Request configuration must include a 'tlsContext' attribute in to specify SSL/TLS settings. TLS context configuration controls certificate validation, supported protocols, and cipher suites for secure HTTPS communication.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:http="http://www.mulesoft.org/schema/mule/http">
  <http:request-config name="HTTP_Request_config">
    <http:request-connection host="api.example.com" port="443" protocol="HTTPS"/>
  </http:request-config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:http="http://www.mulesoft.org/schema/mule/http"
      xmlns:tls="http://www.mulesoft.org/schema/mule/tls">
  <tls:context name="TLS_Context">
    <tls:trust-store path="truststore.jks" password="${tls.password}"/>
  </tls:context>
  <http:request-config name="HTTP_Request_config">
    <http:request-connection host="api.example.com" port="443" protocol="HTTPS" tlsContext="TLS_Context"/>
  </http:request-config>
</mule>
