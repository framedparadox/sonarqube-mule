SMTP configuration must use property placeholders

The 'host', 'port', 'user', and 'password' attributes in must reference property placeholders (e.g., ${smtp.host}) instead of hardcoded values. This ensures email configuration can be changed per environment and credentials remain secure.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:email="http://www.mulesoft.org/schema/mule/email">
  <email:smtp-config name="SMTP_Config">
    <email:smtp-connection host="smtp.gmail.com" port="587" user="user@gmail.com" password="pwd123"/>
  </email:smtp-config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:email="http://www.mulesoft.org/schema/mule/email">
  <email:smtp-config name="SMTP_Config">
    <email:smtp-connection host="${smtp.host}" port="${smtp.port}" user="${smtp.user}" password="${smtp.password}"/>
  </email:smtp-config>
</mule>
