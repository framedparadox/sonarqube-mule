Autodiscovery API ID must use property placeholder

The 'apiId' attribute in must reference a property placeholder (e.g., ${api.id}) instead of a hardcoded value. This enables different API instances across environments to be managed independently in API Manager.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:api-gateway="http://www.mulesoft.org/schema/mule/api-gateway">
  <api-gateway:autodiscovery apiId="12345" apiName="My API" version="1.0"/>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:api-gateway="http://www.mulesoft.org/schema/mule/api-gateway">
  <api-gateway:autodiscovery apiId="${api.id}" apiName="My API" version="1.0"/>
</mule>
