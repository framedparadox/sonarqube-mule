Set Variable doc:name must match the variable name

The doc:name attribute of components should match the variableName attribute for consistency and clarity. This makes flow diagrams and debugging more intuitive by showing the actual variable name being set.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="data-flow">
    <set-variable variableName="myVar" value="#['data']" doc:name="Other Name"/>
  </flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="data-flow">
    <set-variable variableName="myVar" value="#['data']" doc:name="myVar"/>
  </flow>
</mule>
