Application must include MUnit test configuration

Application must contain at least one element indicating the presence of MUnit tests. Unit tests are essential for code quality, regression prevention, and continuous integration. Every Mule application should have corresponding test coverage.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="business-logic">
    <logger message="Processing..."/>
  </flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:munit="http://www.mulesoft.org/schema/mule/munit">
  <munit:config name="munit-config"/>
  <flow name="business-logic">
    <logger message="Processing..."/>
  </flow>
</mule>
