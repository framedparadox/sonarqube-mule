JMS credentials must use property placeholders

The 'username' and 'password' attributes in must reference property placeholders (e.g., ${jms.username}, ${jms.password}) instead of hardcoded values. Hardcoded credentials are a critical security vulnerability.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:jms="http://www.mulesoft.org/schema/mule/jms">
  <jms:config name="JMS_Config">
    <jms:active-mq-connection username="admin" password="secret123"/>
  </jms:config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:jms="http://www.mulesoft.org/schema/mule/jms">
  <jms:config name="JMS_Config">
    <jms:active-mq-connection username="${jms.username}" password="${jms.password}"/>
  </jms:config>
</mule>
