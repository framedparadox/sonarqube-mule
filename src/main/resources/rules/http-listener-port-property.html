HTTP Listener port must use property placeholder

HTTP Listener configuration must use ${https.port} or ${https.private.port} for the port attribute instead of hardcoded values. This allows port configuration to vary across environments (dev, test, prod) without code changes.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:http="http://www.mulesoft.org/schema/mule/http">
  <http:listener-config name="HTTP_config">
    <http:listener-connection host="0.0.0.0" port="8443"/>
  </http:listener-config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:http="http://www.mulesoft.org/schema/mule/http">
  <http:listener-config name="HTTP_config">
    <http:listener-connection host="0.0.0.0" port="${https.port}"/>
  </http:listener-config>
</mule>
