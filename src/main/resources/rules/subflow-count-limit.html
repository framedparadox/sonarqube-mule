Configuration file exceeds maximum subflow count limit

Configuration files should contain fewer than 5 subflows to maintain code organization and readability. Excessive subflows in a single file indicate the need to refactor into separate configuration files.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <sub-flow name="subflow-1"><logger/></sub-flow>
  <sub-flow name="subflow-2"><logger/></sub-flow>
  <sub-flow name="subflow-3"><logger/></sub-flow>
  <sub-flow name="subflow-4"><logger/></sub-flow>
  <sub-flow name="subflow-5"><logger/></sub-flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <sub-flow name="subflow-1"><logger/></sub-flow>
  <sub-flow name="subflow-2"><logger/></sub-flow>
</mule>
