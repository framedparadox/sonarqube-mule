MSSQL database host must use property placeholder

The 'host' attribute in must reference a property placeholder (e.g., ${db.host}) instead of a hardcoded hostname or IP address. Hardcoded database hosts prevent environment portability and expose infrastructure details in code.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:db="http://www.mulesoft.org/schema/mule/db">
  <db:config name="DB_Config">
    <db:mssql-connection host="192.168.1.100" port="1433"/>
  </db:config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:db="http://www.mulesoft.org/schema/mule/db">
  <db:config name="DB_Config">
    <db:mssql-connection host="${db.host}" port="1433"/>
  </db:config>
</mule>
