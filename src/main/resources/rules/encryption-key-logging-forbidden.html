Encryption key must not be exposed in log messages

Logger components must not contain ${mule.key} in their message attribute. Logging encryption keys creates a critical security vulnerability by exposing sensitive credentials in log files. Remove any references to mule.key from logger messages.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="secure-flow">
    <logger message="Encryption key: ${mule.key}"/>
  </flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <flow name="secure-flow">
    <logger message="Processing secure data"/>
  </flow>
</mule>
