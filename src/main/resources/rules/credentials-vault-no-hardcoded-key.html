Secure properties encryption key must use property placeholder

The 'key' attribute in must reference a property placeholder (e.g., ${encryption.key}) instead of a hardcoded value. Hardcoded encryption keys compromise security by exposing them in source code and version control.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:secure-properties="http://www.mulesoft.org/schema/mule/secure-properties">
  <secure-properties:config file="app.properties" key="my-secret-key-12345"/>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:secure-properties="http://www.mulesoft.org/schema/mule/secure-properties">
  <secure-properties:config file="app.properties" key="${encryption.key}"/>
</mule>
