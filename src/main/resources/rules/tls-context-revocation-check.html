TLS context should configure certificate revocation check

TLS context elements should include <tls:revocation-check> to validate certificate revocation status via CRL or OCSP.

Noncompliant Code

<tls:context name="listenerTlsContext">
  <tls:key-store path="${keystore.path}"/>
</tls:context>

Compliant Code

<tls:context name="listenerTlsContext">
  <tls:key-store path="${keystore.path}"/>
  <tls:revocation-check>
    <tls:crl-file path="${crl.path}"/>
  </tls:revocation-check>
</tls:context>
