Database connection URL must use property placeholder

The 'url' attribute in must reference a property placeholder (e.g., ${db.url}) instead of a hardcoded connection string. Hardcoded URLs prevent environment portability and may expose database credentials in code.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:db="http://www.mulesoft.org/schema/mule/db">
  <db:config name="DB_Config">
    <db:generic-connection url="jdbc:mysql://localhost:3306/mydb"/>
  </db:config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:db="http://www.mulesoft.org/schema/mule/db">
  <db:config name="DB_Config">
    <db:generic-connection url="${db.url}"/>
  </db:config>
</mule>
