Subflow names must follow kebab-case naming convention

Subflow names must use lowercase kebab-case format (e.g., 'validate-input', 'transform-response'). This pattern matches lowercase letters, colons, curly braces, and hyphens. Consistent naming conventions ensure code consistency across the project.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <sub-flow name="validateInput"><logger/></sub-flow>
  <sub-flow name="TransformResponse"><logger/></sub-flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <sub-flow name="validate-input"><logger/></sub-flow>
  <sub-flow name="transform-response"><logger/></sub-flow>
</mule>
