APIKit global exception handling strategy must be configured

Application must define a global error handler with APIKit exception types (). This ensures standardized error responses and proper HTTP status code mapping for API errors.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <error-handler>
    <on-error-propagate type="ANY">
      <set-payload value="#{'error': 'Unknown error'}"/>
    </on-error-propagate>
  </error-handler>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <error-handler>
    <on-error-propagate type="APIKIT:BAD_REQUEST">
      <set-payload value="#{'error': 'Bad request'}"/>
    </on-error-propagate>
  </error-handler>
</mule>
