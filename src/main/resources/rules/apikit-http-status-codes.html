APIKit error handlers must define standard HTTP error responses

Global error handler must include on-error-propagate elements for APIKIT:BAD_REQUEST (400), APIKIT:METHOD_NOT_ALLOWED (405), and APIKIT:NOT_FOUND (404). These error handlers ensure proper HTTP status codes are returned for common API validation errors.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <error-handler>
    <on-error-propagate type="ANY">
      <set-payload value="#{'error': 'Error'}"/>
    </on-error-propagate>
  </error-handler>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <error-handler>
    <on-error-propagate type="APIKIT:BAD_REQUEST">
      <set-payload value="#{'error': 'Bad request'}"/>
    </on-error-propagate>
    <on-error-propagate type="APIKIT:NOT_FOUND">
      <set-payload value="#{'error': 'Not found'}"/>
    </on-error-propagate>
    <on-error-propagate type="APIKIT:METHOD_NOT_ALLOWED">
      <set-payload value="#{'error': 'Method not allowed'}"/>
    </on-error-propagate>
  </error-handler>
</mule>
