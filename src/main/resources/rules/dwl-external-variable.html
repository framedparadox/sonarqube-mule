DataWeave variable transformations must use external .dwl files

Transform Message components using should reference external .dwl files via the 'resource' attribute instead of inline scripts. External DataWeave files improve code organization, reusability, testing, and IDE support.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:ee="http://www.mulesoft.org/schema/mule/ee/core">
  <flow name="transform-flow">
    <ee:transform>
      <ee:message>
        <ee:set-variable variableName="result">payload.items</ee:set-variable>
      </ee:message>
    </ee:transform>
  </flow>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:ee="http://www.mulesoft.org/schema/mule/ee/core">
  <flow name="transform-flow">
    <ee:transform>
      <ee:message>
        <ee:set-variable variableName="result" resource="transform.dwl"/>
      </ee:message>
    </ee:transform>
  </flow>
</mule>
