HTTP Request port must use property placeholder

HTTP Request configuration must use ${https.port} or ${https.private.port} for the port attribute instead of hardcoded values. This allows external service port configuration to vary across environments without code changes.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:http="http://www.mulesoft.org/schema/mule/http">
  <http:request-config name="HTTP_Request_config">
    <http:request-connection host="api.example.com" port="443"/>
  </http:request-config>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:http="http://www.mulesoft.org/schema/mule/http">
  <http:request-config name="HTTP_Request_config">
    <http:request-connection host="api.example.com" port="${https.port}"/>
  </http:request-config>
</mule>
