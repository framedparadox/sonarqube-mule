Application must use configuration properties for external resources

Application must include either or to externalize configuration. This ensures credentials, URLs, and environment-specific settings are managed outside the code and can be changed without redeployment.

Noncompliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core">
  <http:listener-config name="HTTP_config" host="0.0.0.0" port="8081"/>
</mule>

Compliant Code

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:secure-properties="http://www.mulesoft.org/schema/mule/secure-properties">
  <secure-properties:config file="app.properties" key="${encryption.key}"/>
  <http:listener-config name="HTTP_config" host="0.0.0.0" port="${http.port}"/>
</mule>
